#### aca se va a simular y comparar el comportamiento de los tres tipos de arboles de busqueda , que seria BST, Splay Tree y red and black(ese esta un poco largo :) ) ademas aca se va a gestionar el proceso de un sistema operativo, con el proceso de busqueda menor que es vrun time, mas tambien que se va a usar el kernel de linux, especificamente el planificador CFS (completely Fair Scheduler), lo que va hacer es que mide cuanto esfuerzo le toma al sistema encontrar el porceso de ejecutarse segun en el tiempo usado

#### en la estructura del codigo va ser dividida en diferentes procesos ,uno de ellos es el modelado de porcesos, que ayuda a crear representantes de tareas en el sistema, cada uno con un identificaor unico y vruntime, otro es el gestion de memoria que ayuda a organizar los procesos en arboles de busqueda para poder encontrar rapidamente cual es el menor tiempo de ejecucion

###Escenario B: Llegada secuencial , dodne los porcesos entran ordenados del 1 al 1000
